Ide yang Masuk v3

Hasil brainstorm bersama
Statistik Kehadiran

Ring % + breakdown (Hadir / Izin-Sakit / Alpa) + mini bar 8 pekan. Data dari AttendanceSummary.

Timer hitung mundur sesi (alur baru)

Hero State A: "Sesi berakhir dalam 23:14" + progress bar. Mini-strip juga tampilkan "Berakhir dalam 23 menit". Mengganti "Mulai 13.00" statis.

Mini-strip sesi aktif (pola Spotify)

State A: strip biru sticky di atas nav — "Pemrograman Mobile · Absen". Muncul HANYA saat ada sesi. Tap → langsung scan. Pembunuh friction utama.

Keputusan desain
Tombol "Ajukan Izin" di empty state DIHAPUS

Atas keputusan user: empty hero lebih bersih tanpa tombol izin. Aksi ajukan izin tetap lewat tab nav "Izin".

Streak di-SKIP

Riset kamu sendiri (mobile-premium-references.md §8) wanti-wanti over-gamifikasi tone-deaf untuk akademik Indonesia. Saya setuju skip.

Jadwal berikutnya di-SKIP

Endpoint cuma return sesi aktif, tidak ada tabel jadwal. Tidak feasible tanpa fitur jadwal baru (effort besar).

Quick action grid TIDAK kembali

Redundan dengan bottom nav (Riwayat/Izin/Profil sudah ada). Diganti aksi kontekstual yang punya nilai tambah.

⚠ Bergantung alur baru (auto-close sesi)
Timer butuh perubahan lintas-layer

Dosen pilih durasi saat buat sesi → ended_at diisi di awal → sesi auto-tutup saat waktu habis. Sentuh DB + web + backend + mobile. Direkomendasikan jadi spec terpisah.

Tetap sediakan tombol "Tutup Sesi" manual

Auto-close = jaring pengaman, BUKAN ganti kontrol dosen. Dosen tetap bisa tutup lebih awal. Pertimbangkan "perpanjang" untuk kelas molor.

Catatan implementasi
Mini bar 8 pekan — perlu agregasi

Ring + breakdown: data langsung. Bar mingguan: hitung dari list riwayat per pekan di app. Mau dipakai atau cukup ring + breakdown?

Grafik = CustomPainter native

Tanpa dependency baru (rule 03 lock). Bukan fl_chart.
