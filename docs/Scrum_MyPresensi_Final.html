BAB 1

User Stories & Persona

Mendefinisikan kebutuhan pengguna

1.1 Latar Belakang Proyek

MyPresensi adalah sistem absensi mahasiswa dengan 3-layer verifikasi (QR Code + GPS + Face Recognition) yang memungkinkan institusi melacak kehadiran secara akurat dan aman. Proyek ini dikerjakan oleh Tim PBL TRPL Politeknik Pertanian Negeri Samarinda menggunakan metodologi Scrum selama 4 sprint dengan durasi masing-masing 2 minggu. Tech stack: Next.js 14 (Web Dashboard), Flutter 3.11 (Mobile App), Supabase (Backend + Auth + Storage + Realtime).

1.2 Persona Pengguna

PersonaDeskripsiKebutuhan Utama
Admin Prodi35-45 tahun, staff administrasi yang mengelola data akademikKelola data mahasiswa/dosen, rekap kehadiran, export laporan
Dosen Pengampu30-50 tahun, mengajar beberapa mata kuliahBuat sesi presensi, monitor kehadiran real-time, approve izin
Mahasiswa20-23 tahun, perlu presensi tiap pertemuanPresensi cepat via HP, lihat riwayat, ajukan izin/sakit

15.5 Kesimpulan

Proyek MyPresensi berhasil mencapai target release candidate dalam 3 sprint (6 minggu), dengan seluruh fitur Must Have, Should Have, dan Could Have terselesaikan. Velocity tim menunjukkan tren positif yang konsisten (42 → 47 → 53 SP), mencerminkan peningkatan produktivitas seiring familiaritas dengan tech stack.

Sistem 3-layer verifikasi (QR + GPS + Face) menjadi diferensiasi utama dibandingkan sistem presensi konvensional, dan kepatuhan terhadap UU PDP menunjukkan kesadaran tim terhadap aspek legal teknologi biometrik.

Sprint 4 saat ini berjalan dengan fokus pada quality assurance dan persiapan demo PBL yang dijadwalkan tanggal 8 Juni 2026.
